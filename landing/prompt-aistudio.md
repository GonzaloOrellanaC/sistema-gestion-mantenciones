Tengo una plataforma web SAAS llamada Sistema de Gestión de Mantenciones (SGM) que sirve para gestionar mantenciones mediante ordenes de trabajo. el flujo del proyecto es el siguiente:

- una persona que tenga una empresa que ejecuta mantenimiento preventivo, correctivo, instalaciones y/o reparaciones de diferentes sistemas que pueden ser mecánicos, electrónicos, eléctricos o cualquier otro (pensando por ejemplo en mantenciones de muebles o todo tipo), puede entrar a la página, registrarse con su empresa
- una vez registrado podrá iniciar sesión en la cual podrá empezar a programar su entorno el cual contempla sucursales de la empresa, activos (que son los elementos a mantener), insumos, repuestos (ambos con identificación de lote de compra y stock) y finalmente usuarios a los que se les asignan las sucursales y los permisos.
- el usuario administrador podrá construir pautas de mantenimiento, que corresponden a los formularios estándar que un usuario de terreno terminará ejecutando cuando se le asigne una orden de trabajo. quiere decir que esa misma pauta sirve para una o varias ordenes de trabajo, ya que se puede repetir para una o varios activos
- cuando esté todo configurado, mediante la creación de ordenes de trabajo el sistema va verificando la existencia del stock de repuestos, insumos necesarios, asigna a los usuarios responsables y como las ordenes de trabajo se ejecutan mediante etapas (por ejemplo creación, asignación, ejecuciones proyectadas, ejecuciones iniciadas realmente, término programado y termino real, etc), se pueden calendarizar las ordenes de tal manera de poder ir haciendo seguimiento
- esta administración se hace mediante una pagina web, mientras que la ejecución de la orden de trabajo se ejecuta desde una aplicación para smartphone o tablet android
- para el control de bodega también se usa la misma app para smartphone o tablet android, ya que se toman fotos y se leen codigos de barraso qr para ir cargando los datos como lotes y precios, con tal de mantener un control de costos de insumos
